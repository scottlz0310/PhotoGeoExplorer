# Issue: 開発版スイッチ（Dev Mode）導入の検討

## 背景
- 手動テストが難しく、開発版へ切り替えるモードが欲しい
- 既存の本番挙動を壊さず、段階的に移行したい
- 変更規模が大きいため、Issue として管理したい

## 目的
- 本番/開発の挙動を明示的に切り替えられる仕組みを用意する
- 段階的な導入と移行が可能な設計にする

## 想定ユースケース
- 新機能やテスト用機能を Dev Mode のみで有効化したい
- 本番ユーザーの体験を維持しながら検証を進めたい

## スコープ（案）
- Dev Mode の切り替え手段
  - 設定画面
  - 起動引数
  - 環境変数
  - 設定ファイル（JSON 等）
- Dev Mode の視認性
  - 画面上のバッジやステータス表示
  - ログに Dev Mode 有効/無効を明記

## 受け入れ条件（案）
- Dev Mode を切り替えても本番挙動がデフォルトで維持される
- Dev Mode の状態が UI またはログで識別できる
- 段階移行のためのトグル設計が明文化されている

## 検討事項
- どの切り替え手段が最も運用しやすいか
- Dev Mode の設定が永続化されるべきか
- 本番配布に Dev Mode が入ることのリスクと対策

## 備考
- 実装に着手する前に、既存のテスト・運用フローとの整合を確認する
