## Issue作成手順

### 方法1: 一括作成スクリプトを使用（推奨）

```bash
# GitHub CLIで認証（初回のみ）
gh auth login

# スクリプトを実行
./scripts/create-v1.5.0-issues.sh
```

### 方法2: 個別にコマンドを実行

詳細は `docs/v1.5.0-issues-creation-guide.md` を参照してください。

## 事前準備

1. **GitHub CLIのインストール**
   ```bash
   # Windows (winget)
   winget install --id GitHub.cli
   
   # macOS
   brew install gh
   
   # Linux
   sudo apt install gh
   ```

2. **GitHub CLIの認証**
   ```bash
   gh auth login
   ```

3. **v1.5.0マイルストーンの確認**
   ```bash
   gh milestone list
   ```
   
   マイルストーンが存在しない場合は、スクリプトが自動で作成を提案します。

## 注意事項

- このリポジトリのAIエージェントは、環境の制約により直接GitHubにIssueを作成できません
- そのため、ユーザーが手動でスクリプトを実行する必要があります
- スクリプトは対話型で、マイルストーンがない場合は作成を提案します

## 実行後の確認

すべてのIssueが作成されたことを確認します：

```bash
# v1.5.0マイルストーンのIssue一覧
gh issue list --milestone v1.5.0

# scottlz0310にアサインされたIssue一覧
gh issue list --assignee scottlz0310

# WebブラウザでIssue一覧を開く
gh issue list --milestone v1.5.0 --web
```

## トラブルシューティング

### 「認証されていません」エラー

```bash
gh auth login
```

を実行して、GitHubアカウントで認証してください。

### 「マイルストーンが見つかりません」エラー

スクリプトが自動で作成を提案しますが、手動で作成する場合：

```bash
gh milestone create v1.5.0 --title "v1.5.0" --description "次期メジャーリリース"
```

### ラベルが存在しない場合

GitHub CLIは存在しないラベルを自動で作成します。特に対応不要です。
